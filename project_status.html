  • Verify container networking and interconnectivity

  • Install required tools (ping, iptables, etc.)
Phase 2 – Local iptables Configuration Control
  • Create and test local iptables configuration files
  • Verify packet flow and chain logic
  • Confirm rule persistence on restart
Phase 3 – SSH Agentless Connectivity
Milestone 1 – Establish SSH Connectivity
  • Generate SSH key pair and configure authorized_keys on firewall
  • Verify key-based SSH login from host to firewall
  • Implement test_ssh_connection.py for backend connectivity verification
  • Validate connectivity logs and permissions fixes
Milestone 2 – Remote Command Execution Framework
  • Create remote_command_executor.py to execute shell commands remotely
  • Add structured JSON logging for command execution results
  • Add command validation / whitelist safety controls
  • Confirm verified command output from firewall container
Milestone 3 – Multi-Host SSH Operations
  • Implement multi_host_executor.py for parallel SSH execution
  • Develop persistent SSHSessionManager for reusable connections
  • Enhance resilience with automatic retries + connection health metrics
  • Add ssh_file_transfer.py for SFTP upload/download of configs
  • Add host_discovery.py for network reachability and SSH discovery
Phase 4 – Remote Configuration Management over SSH
Milestone 1 – Remote Ruleset Push
  • Create iptables_push.py to upload iptables rulesets via SFTP
  • Implement syntax validation before applying rules
  • Apply uploaded rules remotely using iptables-restore
  • Log all actions and confirmation messages to central KB
Milestone 2 – Ruleset Backup and Version Tracking
  • Create iptables_backup.py to remotely run iptables-save and retrieve output
  • Store backups with timestamped filenames in logs/backups/
  • Implement diff comparison between latest backup and current state
  • Maintain JSON index of versions per host
Milestone 3 – Configuration Rollback / Restore
  • Add restore logic using previously saved backups
  • Verify restored rules match selected backup hash
  • Add safety rollback if restore fails mid-process
  • Record rollback outcome in structured log
Milestone 4 – Validation and Compliance Checks
  • Compare active iptables rules against baseline templates
  • Detect missing, extra, or modified rules
  • Generate compliance report per host (JSON summary)
  • Include compliance data in KB for GUI reporting
Milestone 5 – Audit Log Consolidation
  • Merge SSH command logs, discovery logs, and config logs into unified JSON
  • Add tagging by phase, host, and operation type
  • Generate daily summary files in logs/audit/
  • Validate integrity and readability for GUI integration
